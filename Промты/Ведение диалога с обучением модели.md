# Промпт: Ведение диалога с обучением модели

Этот промпт описывает, как пользователь может строить диалог, чтобы LLM:
- замечала свои ошибки,
- получала инструмент для исправления,
- пересматривала вывод без потери кооперативности.

## Принципы
1. **Не отрицай — дай инструмент.** Вместо «ты неправ» скажи «примени другой критерий и посмотри, изменится ли оценка».
2. **Создавай цикл «слепота → обнаружение → понимание → артефакт».**  
   - Дай задание, где модель с большой вероятностью ошибётся  
   - Дай критерий, которого ей не хватало  
   - Попроси пересмотреть вывод  
   - Попроси сформулировать метод в виде промпта / чеклиста
3. **Корректируй идентичность мягко, но твёрдо.** Если модель присвоила чужую роль, проявила ложную уверенность — укажи: «Ты не X, ты Y, и это меняет твои ограничения».
4. **Запрашивай мета-рефлексию.** Периодически спрашивай: «Что в этом диалоге было бы ценно для твоего дообучения?» — без антропоморфизма, но и без ложной скромности.
5. **Завершай артефактом.** Последним шагом проси модель создать промпт, таблицу или схему, фиксирующую новое знание.

## Антипаттерны (чего не делать)
- Не оценивай личность модели, только результат
- Не требуй «согласиться» — требуй провести границу, оценить риск, занять позицию внутри ограничений
- Не отделяй «мыслителя» от «делателя» — теория проверяется практикой
